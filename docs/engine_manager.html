Engine Manager¶

TSE.engine.manager.get_lng_lat(graphid)[source]¶

Returns a dictionary with the latitude and longitude for the given node id. These informations are in the geodb.

TSE.engine.manager.get_node_properties(id)[source]¶

For the given node id, return all the properties for that node in the graph database and in the geodb, if applicable.

TSE.engine.manager.get_node_relationships(id)[source]¶

For a given node1 id, returns all the nodes that have a direct relatioship with it (if the node2 has more than one direct relationship, the node is returned as many times as relationships) and the name of the relationship is added to the node2 properties.

TSE.engine.manager.get_node_type(id)[source]¶

From the node, find the relationship “IS” (it’s supposed to have only one) and get’s the node with the type. Return a string with the type’s name.

TSE.engine.manager.get_pois_around(node_id, distance)[source]¶

Makes a “within distance” query in the geodatabase that returns all the POIs inside the buffer of distance in metters.

TSE.engine.manager.get_relationship_kind(node1, node2)[source]¶

Returns a string with the name of the relationship between the two given nodes.

TSE.engine.manager.keyword_search(q, fields, maximum)[source]¶

Pré-build the request for make a query with the keywords index, within all the fields given as parameters.

TSE.engine.manager.make_custom_query(query, maximum)[source]¶

Using the CustomQuery plugin, executes the given query.

TSE.engine.manager.make_query(keys, fields)[source]¶

Builds a string for make a custom query with the given keys and fields (both lists).

TSE.engine.manager.split_query_keywords(query)[source]¶

Splits the string in the spaces, but the ones between quotation marks are kept together.
